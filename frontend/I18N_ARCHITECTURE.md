## Component Integration Patterns

### Pattern 1: Server Component with Single Namespace

```typescript
// Simple, performant - only loads one namespace
import { useTranslations } from 'next-intl'

export default function SimpleButton() {
  const t = useTranslations('common')
  return <button>{t('save')}</button>
}
```

**Flow:**
```
Component â†’ useTranslations('common') â†’ Load common namespace â†’ t('save') â†’ "Speichern"
```

### Pattern 2: Client Component with Multiple Namespaces

```typescript
'use client'
import { useTranslations } from 'next-intl'

export default function LoginForm() {
  const t = useTranslations('auth.login')
  const tCommon = useTranslations('common')
  const tErrors = useTranslations('errors')

  return (
    <form>
      <h1>{t('title')}</h1>
      <input placeholder={t('email')} />
      <button>{tCommon('submit')}</button>
    </form>
  )
}
```

**Flow:**
```
Component â†’ useTranslations('auth.login') â†’ Load auth.login namespace
         â†’ useTranslations('common')      â†’ Load common namespace
```
